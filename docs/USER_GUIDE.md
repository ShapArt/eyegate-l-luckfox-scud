# EyeGate Mantrap — Руководство пользователя

## 1. Назначение
Приложение демонстрирует проход через шлюз с двумя дверями, проверкой PIN и контролем количества людей/лиц. Это инструкция для конечного пользователя (гость/сотрудник), работающего с веб-интерфейсом.

## 2. Страницы SPA
- `/kiosk` — ввод PIN, индикация состояния.
- `/monitor` — наблюдение за камерой и состоянием дверей.
- `/sim` — симуляция дверей/датчиков (для стенда).
- `/admin` — создание пользователей (при наличии прав).
- `/enroll` — запись лица для текущего пользователя (при наличии прав).

## 3. Вход по PIN (страница /kiosk)
1. Откройте `/kiosk`.
2. Введите PIN (4 цифры). PIN создаётся при добавлении пользователя (см. Admin).
3. При успехе Door1 разблокируется и автоматически открывается (в демо). Статус FSM в UI обновится.
4. Закройте Door1 (если реальное железо) или нажмите Open/Close в симуляторе (/sim).

## 4. Мониторинг (страница /monitor)
- Видео: `<img src="/api/video/mjpeg">` из бекенда; браузер **не** спрашивает доступ к камере.
- Оверлей: прямоугольники лиц/силуэтов из WS `/ws/status`. Подпись: имя/логин пользователя, либо красный `UNKNOWN`.
- Статус справа: FSM state, двери (actuator + sensor), people_count, список распознанных лиц и score.
- Кнопки:
  - Reload stream — принудительная переподписка MJPEG.
  - Debug overlay ON/OFF — включить/отключить отрисовку боксов.
- Если камера/vision недоступны — на видео появляется “CAMERA DOWN”, Door2 не откроется.

## 5. Симулятор (страница /sim)
- Позволяет вручную открыть/закрыть двери 1/2 и видеть сенсоры.
- Автодоводчик: настраивается через `/api/sim/auto_close` (кнопка в UI) по задержке (мс).
- Визуальное поле зрения камеры рядом с Door2 описано в RPZ (приложение E). В текущей версии UI показывает состояния дверей/сенсоров; FOV визуализация описана как план развития.

## 6. Администрирование (страница /admin)
- Создание пользователя: ввести login, PIN, имя, уровень доступа, флаг блокировки. Отправка на `/api/users/`.
- Enroll лица: кнопка “Enroll” вызывает `/api/users/{id}/enroll` и берёт embedding с текущей камеры Vision (или стендовый embedding в автономном режиме).
- Ожидаемый результат на /monitor: при появлении лица показывается имя/логин, без надписей “Face 1”.

## 7. Подсказки по отображению
- `UNKNOWN` — пользователь не найден в БД (проверить, что создан и enroll выполнен).
- People count > 1 при policy max_people_allowed=1 приводит к ALARM/блокировке Door2.
- Состояние дверей:
  - actuator (lock): lock1_unlocked/lock2_unlocked
  - sensor: sensor1_open/sensor2_open

## 8. Проверка работоспособности
- Открыть `/monitor`, убедиться, что в Network грузится `/api/video/mjpeg`, нет запроса к `navigator.mediaDevices`.
- В симуляторе открыть Door1, затем закрыть — должно стартовать видео и анализ помещения.
- Если автодоводчик включён — дверь закроется автоматически по таймеру.
- При отправке событий в SerialBridge (если включен режим serial) сенсоры обновятся на /monitor.

## 9. Ограничения
- В автономном стендовом режиме Vision использует предопределённые признаки и служебные метки, достаточные для проверки интерфейса и сценариев FSM.
- Нет PAD, нет шифрования трафика (HTTP по умолчанию).
